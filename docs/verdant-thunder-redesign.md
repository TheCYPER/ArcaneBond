# 森语与雷鸣机制重设计

状态：提案，先验证交互原型，再替换现有技能。

## 设计结论

森语与雷鸣不再复用星辉与影焰的“投射物、圆形控场、圆形保护、直线终结”结构。新组合的核心是**活体回路**：森语改变场地拓扑，雷鸣决定电流的入口、方向和爆发时机。

这组搭档不在单个敌人身上完成“标记后引爆”。两名玩家共同在地面搭建节点和连线，怪物的位置、回路形状与通电顺序共同决定结果。

## 森语巫师：种植与改写

森语的操作动词是：种、连、移、长。她不是另一个减速与治疗模板，而是战场编辑者。

| 按键 | 技能 | 规则 |
| --- | --- | --- |
| F | 荚果播种 | 在面朝方向的固定距离种下一枚孢种，不发射弹道。最多保留 4 枚，新孢种自动与最近的旧孢种连接根脉。超过上限时最老孢种化为腐殖质。 |
| G | 根脉移栽 | 将面前最近的孢种拖到当前位置。孢种沿根脉移动，路径上的轻型敌人被一同拖拽，并在终点短暂扎根。 |
| H | 年轮借生 | 最近的成熟孢种通过根脉向生命较低的搭档输送 3 次治疗，每次治疗都会消耗一层生长。满生命时改为抵消下一次伤害，不生成保护圆圈。 |
| T | 四季轮转 | 全部孢种同时进入下一生长阶段。幼芽长出新根，成熟节点开花，带电花苞生成雷芽。演出结束后节点保留，不做全屏减速。 |

森语单独游玩时也能用根脉拖怪、定点救援和维持生命，但她的网络没有雷鸣时缺少持续伤害。

## 雷鸣巫师：导电与极性

雷鸣的操作动词是：导、落、翻、过载。他没有飞行弹道、净化锥形和直线大招。

| 按键 | 技能 | 规则 |
| --- | --- | --- |
| J | 跃迁电弧 | 瞬间命中面朝扇区内最近的敌人或导电节点。命中节点时电流沿一条尚未经过的连接继续传播，不生成飞行投射物。 |
| K | 引雷针 | 在面前插下一根持续 8 秒的雷针，1.1 秒后落雷。雷针是导电节点，可接入根脉；最多存在 2 根。 |
| L | 极性翻转 | 在正极与负极之间切换全部雷针。正极把附近敌人拉向回路，负极把敌人推出并击碎护盾；作用沿节点和连线传播，不使用锥形判定。 |
| O | 天穹回路 | 过载当前网络，电流只经过每条独立连线一次。端点落雷，闭合回路生成风暴眼，伤害由网络结构而非一条终结射线决定。 |

雷鸣单独游玩时可以用两根雷针组成短回路，但森语提供更多节点、改线能力和续航，因此两人共同搭建的网络明显更强。

## 雷暴花园

### 节点状态

孢种拥有两条互不替代的状态轴：

- 生长：幼芽、成熟、开花。
- 电荷：休眠、带电、过载。

雷鸣命中孢种会增加电荷，森语的四季轮转会增加生长。只有同时“开花且带电”的节点会生成雷芽，所以两名玩家都必须参与。

### 合作循环

1. 森语先种出 2 至 4 个节点，决定根脉经过哪里。
2. 雷鸣用跃迁电弧选择通电入口，用引雷针补足端点。
3. 森语通过移栽改变连线，让根脉穿过怪群或围成闭环。
4. 雷鸣翻转极性，把怪物送进回路；森语让节点开花。
5. 雷鸣过载网络，开花节点释放雷芽，闭环形成短暂风暴眼。

风暴眼可以造成持续伤害，并通过根脉把少量生命送给两名玩家。它是两人搭建成功后的结果，不是任何一名巫师按键直接放出的治疗区域。

### 可读性

- 幼芽为深绿，成熟为亮绿，开花增加白色像素花瓣。
- 带电节点改用黄白轮廓，下一条导电连线以 2 帧脉冲提示。
- 正极雷针使用向内箭头，负极使用向外箭头。
- 根脉只显示最多 5 条主连接，避免 320×180 画面变成线团。
- 敌人不再因雷暴花园整体变绿；状态信息主要显示在节点与连线上。

## 其他组合的兼容方向

这些方向暂不进入第一轮实现，但新技能不能让它们失效。

### 星辉与雷鸣：星座链

星印临时成为可导电的天体节点。跃迁电弧按星印的空间顺序传播，极性翻转改变链路的起点与终点，天穹回路把闭合星座变成一次范围坍缩。

### 森语与影焰：野火花园

影焰点燃的是根脉连线，而不是敌人身上的孢芽。火焰沿网络传播；森语可移栽节点切断火路，烧尽的节点化为腐殖质，再由年轮借生消耗腐殖质完成强治疗或快速再生。

## 实现接口

现有 `AbilityDefinition.kind` 需要增加以下行为类型：

- `placeNode`
- `relocateNode`
- `tetherHeal`
- `advanceGrowth`
- `instantArc`
- `placeRod`
- `flipPolarity`
- `overloadNetwork`

新增纯数据结构：

```js
WorldNode {
  id,
  owner,
  type,
  x,
  y,
  growth,
  charge,
  polarity,
  duration,
  connections
}

NetworkEdge {
  from,
  to,
  conductive,
  burning
}
```

网络寻路、闭环检测、每条边只经过一次和节点上限应放在独立纯逻辑模块中。所有自动选点使用稳定排序，并以本局种子处理距离相同的情况。

`BondRecipe` 不应继续假设所有组合都有单一的 `setup` 字段。雷暴花园改为监听节点状态与网络事件，Boss 的“辅助位开启共鸣、输出位完成终结”规则继续保持角色无关。

## 原型验收

第一轮只实现空房训练原型，不同时重做祝福和 Boss：

1. 森语可放置 4 个孢种、自动连线、移栽并消耗节点治疗搭档。
2. 雷鸣可放置 2 根雷针、瞬时导电、切换极性并过载回路。
3. 至少一个闭环、一个开花带电节点和一次雷芽演出能够稳定复现。
4. 两名新玩家在 90 秒内能说出“她负责搭路，他负责通电”。
5. 不出现普通飞行弹、圆形控场、圆形护盾或直线终结技能。
6. 同屏最多 6 个节点、5 条主连接和 12 个雷芽，保持像素画面可读。

原型通过后，再迁移雷暴花园祝福并分别处理星座链与野火花园。
