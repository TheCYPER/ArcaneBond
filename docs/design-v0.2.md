# 《双生秘法：Arcane Bond》本地双人 Demo 设计文稿 v0.2

## 1. 项目定位

这是一个**本地双人同屏网页游戏 Demo**。

它不需要联网、不需要账号、不需要服务器、不需要房间码。玩家只需要打开一个 HTML 页面，就可以在同一台电脑上用同一套键盘进行双人合作战斗。

游戏目标不是做成完整商业游戏，而是做一个可以和妹妹一起玩的轻量 Demo。核心重点是：

1. 两名玩家在同一张地图里合作。
2. 两名玩家使用不同按键操作。
3. 两名玩家能力互补，不能完全各打各的。
4. 游戏有巫师、怪物、魔法、Boss 和配合感。
5. 操作简单，上手快，适合本地娱乐和后续扩展。

## 2. 核心玩法一句话

哥哥和妹妹分别扮演两名不同类型的巫师，一个负责控制、保护、标记敌人，另一个负责爆发、破盾、终结怪物。两人必须在同一套键盘上配合，才能打败怪物和 Boss。

### 原方案

### 推荐技术

Phaser 的键盘系统支持添加指定按键、监听键盘事件、在 update 循环里检测按键状态，这非常适合做同屏双人操作。

## 4. 玩家人数

固定 2 人。

不设计单人模式。
不设计在线模式。
不设计 AI 队友。

因为这个 Demo 的重点就是两个人坐在一起合作。

## 5. 屏幕与视角

建议采用**单屏俯视角**。

两名玩家始终在同一个画面内，不能走得太远。

如果两人距离太远，可以有两种处理方式：

### 方案 A：空气墙限制

当两名玩家距离过远时，角色无法继续远离对方。

优点：实现简单，适合 Demo。
缺点：自由度稍低。

### 方案 B：共享摄像机

摄像机跟随两名玩家的中心点，地图可以稍微大一些。

优点：表现更好。
缺点：实现稍复杂。

MVP 建议采用方案 A：限制两人最大距离，让画面始终简单清楚。

## 6. 双人键盘操作设计

这是本版最重要的修改。

两名玩家共用一套键盘，因此按键必须避免冲突、避免手的位置打架、避免需要太多同时按键。

## 6.1 玩家 A：星辉巫师，键盘左侧

适合坐在键盘左边的人操作。

移动：

W：上
A：左
S：下
D：右

技能：

F：普通星光弹
G：星环禁锢
H：守护结界
T：终极技能，星穹封锁
E：救援 / 互动

## 6.2 玩家 B：影焰巫师，键盘右侧

适合坐在键盘右边的人操作。

移动：

方向键 ↑：上
方向键 ←：左
方向键 ↓：下
方向键 →：右

技能：

小键盘 1：普通影焰箭
小键盘 2：裂魂火柱
小键盘 3：破咒烈焰
小键盘 0：终极技能，深红审判
小键盘 Enter：救援 / 互动

## 6.3 无小键盘备用方案

如果电脑没有小键盘，尤其是笔记本电脑，可以使用右侧字母区：

移动：

I：上
J：左
K：下
L：右

技能：

U：普通影焰箭
O：裂魂火柱
P：破咒烈焰
右 Shift：终极技能
Enter：救援 / 互动

## 6.4 推荐最终操作方案

为了适配更多电脑，建议 Demo 默认使用：

玩家 A：

WASD 移动
F / G / H / T / E 操作

玩家 B：

方向键移动
J / K / L / O / P 操作

这样即使没有小键盘，也可以直接玩。

最终建议如下：

| 功能      | 玩家 A：星辉巫师 | 玩家 B：影焰巫师 |
| ------- | --------- | --------- |
| 上       | W         | ↑         |
| 下       | S         | ↓         |
| 左       | A         | ←         |
| 右       | D         | →         |
| 普通攻击    | F         | J         |
| 技能 1    | G         | K         |
| 技能 2    | H         | L         |
| 终极技能    | T         | O         |
| 互动 / 救援 | E         | P         |

## 7. 按键设计原则

### 7.1 每人控制区域分开

玩家 A 使用键盘左侧。
玩家 B 使用方向键和右侧字母。

这样两个人的手不会经常碰到一起。

### 7.2 不使用鼠标

本地双人共用一台电脑时，鼠标只能方便一个人使用。为了公平和简单，本 Demo 不使用鼠标控制攻击方向。

### 7.3 攻击方向自动判定

因为不使用鼠标，攻击方向可以这样设计：

默认朝角色当前移动方向攻击。
如果角色没有移动，则朝上一次移动方向攻击。
范围技能释放在角色前方固定距离。
护盾技能释放在自己脚下或队友附近。

这样两名玩家只需要键盘就能完成所有操作。

## 8. 角色设计修改

## 8.1 玩家 A：星辉巫师

定位：控制、保护、标记、救援。

星辉巫师适合更稳一点的玩家。他的任务不是疯狂打伤害，而是观察场面，控制怪物，保护队友，制造连携机会。

### 普通攻击：星光弹

按键：F

效果：

向当前朝向发射一枚星光弹。
造成少量伤害。
命中敌人后添加 1 层“星印”。
带有星印的敌人被影焰巫师攻击时会爆炸。

### 技能 1：星环禁锢

按键：G

效果：

在角色前方生成一个星辉法阵。
法阵内的敌人被减速或短暂定身。
适合配合影焰巫师的火柱技能。

### 技能 2：守护结界

按键：H

效果：

在星辉巫师脚下生成一个圆形护盾区域。
站在护盾里的玩家受到伤害降低。
护盾持续几秒。
适合保护影焰巫师蓄力施法，或者救援倒地队友。

### 终极技能：星穹封锁

按键：T

效果：

短时间内全场怪物减速。
Boss 核心暴露。
影焰巫师在此期间使用终极技能可以造成大量伤害。

### 互动 / 救援

按键：E

效果：

靠近倒地队友时，长按或持续站在旁边可以救援。
救援完成后，队友恢复部分生命值。

## 8.2 玩家 B：影焰巫师

定位：输出、破盾、爆发、终结。

影焰巫师适合喜欢攻击的玩家。他的伤害高，但更脆弱，需要星辉巫师保护。

### 普通攻击：影焰箭

按键：J

效果：

向当前朝向发射一枚影焰箭。
造成中等伤害。
如果命中带有星印的敌人，会触发星焰爆裂。

### 技能 1：裂魂火柱

按键：K

效果：

在角色前方生成火柱。
短暂延迟后爆发。
对范围内敌人造成高伤害。
如果敌人被星环禁锢控制，几乎可以稳定命中。

### 技能 2：破咒烈焰

按键：L

效果：

发射一道短距离破咒火焰。
可以移除怪物护盾。
可以解除队友身上的诅咒。
对护盾怪非常关键。

### 终极技能：深红审判

按键：O

效果：

向当前方向释放一道高伤害光束。
普通情况下伤害较高。
如果 Boss 正处于星穹封锁状态，则对 Boss 核心造成巨额伤害。

### 互动 / 救援

按键：P

效果：

靠近倒地队友时进行救援。

## 9. 本地双人合作重点

这版游戏必须避免变成“两个人各打各的”。因此每一种怪物都应该鼓励一种配合方式。

## 9.1 小怪配合

腐化小魔数量多。

推荐打法：

星辉巫师用星环禁锢聚怪。
影焰巫师用裂魂火柱清怪。

## 9.2 快速怪配合

影爪兽会优先冲向影焰巫师。

推荐打法：

星辉巫师提前用星环禁锢拦住。
影焰巫师后退并使用影焰箭输出。

## 9.3 护盾怪配合

咒甲守卫拥有黑暗护盾。

推荐打法：

影焰巫师用破咒烈焰破盾。
星辉巫师给它叠星印。
影焰巫师引爆星印造成高伤害。

## 9.4 诅咒怪配合

噬魔幽魂会让一名玩家进入诅咒状态。

推荐打法：

被诅咒玩家后撤。
另一名玩家保护。
影焰巫师用破咒烈焰解除诅咒。

## 10. 关卡结构：本地 Demo 版本

因为只是本地 Demo，内容应该控制在 5–10 分钟内。

建议做一个完整小关卡，而不是多个大关。

## 10.1 Demo 流程

### 第一段：操作教学区

目标：

两名玩家移动到两个发光法阵上。
系统提示各自按键。
玩家测试普通攻击。

出现 3 只腐化小魔。

教学内容：

移动
攻击
同屏合作
星印引爆

### 第二段：合作战斗区

出现两波怪物。

第一波：

腐化小魔 x 6
影爪兽 x 2

要求：

星辉巫师控制怪物。
影焰巫师输出。

第二波：

咒甲守卫 x 2
腐化小魔 x 4

要求：

影焰巫师破盾。
两人集火。

### 第三段：救援教学区

设计一个高压小战斗，让其中一名玩家可能倒地。

系统提示：

“队友倒地！靠近并按 E / P 救援！”

目标：

让玩家理解倒地不等于立即失败，必须互相救援。

### 第四段：Boss 战

Boss：裂隙大巫妖。

Boss 分两个阶段即可，不需要三阶段。

第一阶段：

召唤小怪
发射魔法弹
周期性生成黑暗护盾

第二阶段：

Boss 核心封闭
星辉巫师需要按 T 暴露核心
影焰巫师需要按 O 使用深红审判
完成两次终极连携后击败 Boss

## 11. 胜利与失败条件

## 11.1 胜利条件

击败 Boss。
两名玩家至少有一人存活。
封印裂隙核心。

## 11.2 失败条件

两名玩家同时倒地。
或者一名玩家倒地后，另一名玩家没有在 10 秒内救援成功。
或者 Boss 战中裂隙能量充满。

## 12. 难度设计

因为是和妹妹一起玩，难度应该偏轻松，不要太硬核。

建议：

普通怪移动速度较慢。
技能冷却时间短。
玩家生命值较高。
救援时间 2 秒。
倒地后有 10 秒救援时间。
Boss 攻击前有明显红圈提示。
每次成功连携都给明显反馈。

## 13. 亲友同玩体验设计

这是本地 Demo 的特殊重点。

## 13.1 明确的角色分工

可以在开始界面写：

玩家 A：保护型巫师，适合喜欢控制和救人的玩家。
玩家 B：攻击型巫师，适合喜欢打伤害的玩家。

## 13.2 语音沟通提示

“我控住了，你放火柱！”
“我被诅咒了，帮我解除！”
“Boss 核心出现了，准备大招！”

## 13.3 失败不惩罚

失败后不要回到最开始，可以提供：

降低难度再来一次

## 13.4 可爱化处理

怪物不要太吓人。
Boss 可以更像“被污染的老巫师”而不是恐怖怪物。
失败提示不要写“死亡”，可以写“魔力耗尽”。
救援可以叫“唤醒同伴”。

## 19. 开发实现建议

## 19.1 文件结构

建议保持简单：

index.html
src/main.js
src/scenes/BootScene.js
src/scenes/GameScene.js
src/entities/Player.js
src/entities/Enemy.js
src/entities/Boss.js
src/systems/InputManager.js
src/systems/CombatSystem.js
assets/images
assets/audio

## 19.2 输入系统

用 Phaser 的 keyboard input 管理按键。

玩家 A 和玩家 B 分别绑定不同按键。

每一帧读取：

玩家 A 是否按下 W/A/S/D
玩家 A 是否按下 F/G/H/T/E
玩家 B 是否按下方向键
玩家 B 是否按下 J/K/L/O/P

Phaser 支持通过键盘插件添加 Key 对象并在 update 循环中轮询，这很适合本地双人键盘控制。

## 19.3 攻击方向

每个玩家保存 lastDirection。

当玩家移动时，更新 lastDirection。
当玩家攻击时，按照 lastDirection 生成法术。
如果刚开始没有方向，默认向下或向右。

## 19.4 碰撞与伤害

玩家法术碰到怪物：

减少怪物生命值。
如果是星光弹，添加星印。
如果是影焰箭且怪物有星印，触发爆炸。

怪物碰到玩家：

减少玩家生命值。
短暂无敌 0.5 秒，避免连续掉血过快。

## 19.5 本地运行方式

开发阶段可以用本地服务器运行：

例如 VS Code Live Server
或使用简单命令启动本地 HTTP 服务

最终可以打包成一个文件夹。你和妹妹只要打开页面就能玩。

## 20. 本版最终设计目标

本地 Demo 不追求复杂，而追求好玩、清楚、能配合。

最重要的体验是：

1. 妹妹能很快知道自己按什么。
2. 两个人坐在一起不会抢键盘。
3. 每个角色都有存在感。
4. 打怪时会自然喊出“我控住了！”“你快打！”“快救我！”。
5. Boss 被双人大招击败时有明显爽感。
