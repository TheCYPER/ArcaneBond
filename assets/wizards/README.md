# 巫师像素资源绘制指南

每位巫师拥有独立目录：`star`、`ember`、`verdant`、`thunder`。

目录内容：

- `sheet.png`：游戏实际读取的 17 帧精灵表。
- `portrait.png`：HUD 与选角界面使用的正面头像。
- `frames.json`：帧编号、标签与尺寸元数据。
- `legacy.png`：旧版 6 帧资源，仅作造型和配色参考。

## 精灵表规格

- 单帧：16×16 RGBA。
- 总帧数：17。
- 相邻帧之间：1 像素纯黑分隔线。
- 整张精灵表：288×16。
- 导出时禁用缩放与插值，不要删除黑色分隔线。

| 帧编号 | 内容 |
| --- | --- |
| 0–1 | 向上移动，两帧 |
| 2–3 | 向下移动，两帧 |
| 4–5 | 向左移动，两帧 |
| 6–7 | 向右移动，两帧 |
| 8–9 | 向上施法，两帧 |
| 10–11 | 向下施法，两帧 |
| 12–13 | 向左施法，两帧 |
| 14–15 | 向右施法，两帧 |
| 16 | 倒地静态帧 |

在 Aseprite 中可以把网格设置为 `17×16`，每格最后一列是黑色分隔线，实际角色绘制区域始终为左侧 `16×16`。完成正面造型后，也要同步更新同目录的 `portrait.png`。

运行 `npm run assets:wizards` 会重新生成占位图，因此手绘开始后不要执行这个命令覆盖成品。
